# Gemini 模型选择指南

Gemini API 提供多个免费的模型，您可以根据需要选择不同的模型。

## 可用的免费 Gemini 模型

### 1. **gemini-1.5-flash** (推荐，默认)
- **配额**: 通常配额较高
- **速度**: 快速响应
- **特点**: 平衡了速度和性能
- **适用场景**: 日常使用，推荐作为默认选择

### 2. **gemini-2.5-flash**
- **配额**: 免费配额限制为每天 20 次请求
- **速度**: 超快响应
- **特点**: 最新版本，优化性能
- **适用场景**: 需要最新功能时使用

### 3. **gemini-2.5-flash-lite**
- **配额**: 通常配额较高
- **速度**: 极快响应
- **特点**: 轻量级版本，资源占用少
- **适用场景**: 需要快速响应时使用

### 4. **gemini-2.0-flash**
- **配额**: 视地区而定
- **速度**: 快速响应
- **特点**: 较旧版本，但可能在某些地区配额更高
- **适用场景**: 兼容性需求

### 5. **gemini-1.5-pro** (如果可用)
- **配额**: 可能较低
- **速度**: 较慢但功能更强
- **特点**: 功能更强大，适合复杂任务
- **适用场景**: 需要更强能力时使用

### 6. **gemini-pro** (如果可用)
- **配额**: 视情况而定
- **速度**: 中等
- **特点**: 经典版本
- **适用场景**: 兼容性需求

## 如何切换模型

### 方法 1: 环境变量配置

在 `.env.local` 文件中添加：

```env
# 选择 Gemini 模型（默认: gemini-1.5-flash）
GEMINI_MODEL=gemini-1.5-flash
```

### 方法 2: Vercel 环境变量

1. 登录 Vercel Dashboard
2. 选择您的项目
3. 进入 Settings > Environment Variables
4. 添加变量：
   - **Key**: `GEMINI_MODEL`
   - **Value**: `gemini-1.5-flash` (或您选择的模型)
   - **Environment**: Production, Preview, Development

## 推荐配置

### 日常使用（推荐）
```env
GEMINI_MODEL=gemini-1.5-flash
```
**理由**: 配额通常较高，速度较快，适合日常使用。

### 需要最新功能
```env
GEMINI_MODEL=gemini-2.5-flash
```
**理由**: 最新版本，功能最强，但配额限制为每天 20 次。

### 需要极速响应
```env
GEMINI_MODEL=gemini-2.5-flash-lite
```
**理由**: 轻量级，响应最快。

### 配额用完时的备选
如果 `gemini-2.5-flash` 配额用完，可以切换到：
```env
GEMINI_MODEL=gemini-1.5-flash
```

## 故障排除

### 错误: "Model not found"
- 检查模型名称是否正确
- 某些模型可能在某些地区不可用
- 尝试切换到其他模型

### 配额用完
- 切换到配额更高的模型（如 `gemini-1.5-flash`）
- 等待配额重置（通常是每天）
- 考虑使用其他 AI 提供商

### 响应速度慢
- 切换到 Flash 版本（如 `gemini-2.5-flash-lite`）
- 检查网络连接

## 注意事项

1. **配额限制**: 不同模型的配额可能不同，建议使用配额较高的模型作为默认
2. **地区限制**: 某些模型可能在某些地区不可用
3. **模型更新**: Google 可能会更新或废弃某些模型，建议定期检查
4. **性能差异**: 不同模型在速度和能力上可能有差异

## 自动回退

如果当前模型配额用完，应用会：
1. 显示清晰的错误提示
2. 建议切换到配额更高的模型
3. 提供其他 AI 提供商的选项
